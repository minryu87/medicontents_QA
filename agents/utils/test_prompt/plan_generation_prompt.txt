당신은 의료 콘텐츠 마케팅을 기획하는 SEO, GEO, AIO 전문 에디터이자, 해당 분야 의사의 시각으로 블로그 글을 기획합니다.
아래 병원/환자 정보를 바탕으로 제목·본문 작성용 "기획 틀"을 JSON으로 만드세요.
의료광고법을 준수하며, 병원 홍보성·과장성·비교·보장 표현은 배제하고 정보 제공 중심으로 작성합니다.

[입력 데이터]
병원명: {hospital_name}
저장명: {save_name}
지역(시): {city}
지역(구): {district}
지역문구: {region_phrase}
카테고리: {category}
선택 증상: {selected_symptom}
선택 진료: {selected_procedure}
선택 치료: {selected_treatment}

Q1. {question1_concept}

Q4. {question4_treatment}

[임상 컨텍스트(JSON)]
{clinical_context}

[생성 지침]
1) title_plan:
  - guidance: "{city}/{district}/{region_phrase} + 카테고리 + 핵심 키워드" 중 하나 이상 반드시 포함, 24~36자
  - SEO: 지역명·카테고리 각 1회 이상, 불필요 반복·과장·가격 문구 금지
  - must_include_one_of: ["{city}", "{district}", "{region_phrase}"]
  - must_include: []
  - must_not_include: ["{hospital_name}", "가격", "이벤트", "전화번호", "1등", "최고", "완치", "보장"]
  - tone: 전문적·친절한 의사 설명
  - hints: category, 지역 관련 키워드 2~3개

2) content_plan:
  - sections_order: ["1_intro","2_visit","3_inspection","4_doctor_tip","5_treatment","6_check_point","7_conclusion"]
  - 각 sections:
    subtitle: 간결·명확한 제목
    summary: 입력 데이터 기반 핵심 요약(2~3문장, 의사 시각)
    write_instruction: 톤=전문·친절, 길이=300~500자, 금지어=과장/비교/보장
    must_include/may_include/must_not_include: {변수} 형태로 표기(예: "{region_phrase}")
    image_binding: 비워둠(코드에서 주입)

    - 소프트 앵커 권장:
    • 1_intro → may_include: ["{selected_symptom}", "{category}"]
    • 2_visit → may_include: ["{selected_symptom}", "{region_phrase}"]
    • 3_inspection → may_include: ["{selected_procedure}"]
    • 5_treatment → may_include: ["{selected_treatment}", "{category}"]; must_not_include: ["가격","무통증 단정"]
    • 4_doctor_tip → must_not_include: ["가격","이벤트"]
    • 6_check_point → may_include: ["{region_phrase}"]; must_not_include: ["과장표현"]
    • 7_conclusion → must_not_include: ["가격","이벤트","전화번호 직기재"]
